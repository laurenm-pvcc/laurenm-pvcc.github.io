All About GitHub

  1. Q. What is a version control system?
    • A. A version control system tracks changes to a file or set of files over time.
  2. Q. What is Git?
    • A. Git is a DevOps tool used for source code management. It is a free open-source VCS.
  3. Q. What is GitHub?
    • A. It is a web-based interface that uses Git. Fun Fact: It was officially founded as a company after changing its name from Logically Awesome.
  4. Q. What is the difference between Git and GitHub?
    • A. While Git is a tool that's used to manage multiple versions of source code edits that are then transfered to files in a Git repository, GitHub serves as a location for uploading copies of a Git repository.
  5. Q. Who started GitHub and how was it started?
    • A. The founders of GitHub are Tom Preston-Werner, P.J. Hyatt, Chris Wanstrath, and Scott Chacon. They created it for their own work, to collaborate on programming projects.
  6. Q. What company owns it now?
    • A. Microsoft
  7. Q. How much does a GitHub account cost?
    • A. Anyone can join for free. There is also a Team plan for $44 a year, or $4 per user/month. Also an Enterprise plan for $231 a year, or $21 per user/month. There are also additional add-ons for your desired plan level.
  8. Q. What is the Octocat?
    • A. It is the mascot of the source-code hosting service GitHub.

    Git/GitHub Terms

    • Repository
      • A directory that stores all of the files and folders you used to build the project and it also stores the changes made to the project. Also called a "repo".
    • Commit
      • a change that you bring to your program, it can be adding, removing, modifying code or files from your project.
    • Fork
      • Instead of a single server-side repository, forking gives every developer a server-side repository.
    • Push
      • Push changes from local to remote.
    • Pull requests
      • They provide a user-friendly web interface for discussing proposed changes.
    • Workflows
      • Centralized Workflow
        • For developers already comfortable with Subversion. It gives you the benefits of Git without having to adapt to an entirly new enterprise.
      • Feature branch
        • It builds on the Centralized Workflow. Enables the use of pull requests as a means to discuss changes before they are integrated into the official project.
      • Gitflow Workflow
        • It streamlines the release cycle.
    • Issues
      • They let you track your work on GitHub, where development happens.
    • Raw button
      • With the raw view, you can view or copy the raw content of a file without any styling.
    • Blame Button
      • The git blame command is used to examine the contents of a file line by line and see when each line was last modified and who the author of the modification was.
